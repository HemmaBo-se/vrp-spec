Implement VRP in 5 minutes

Ship your own host-domain signer. No central registry. No approval process. No marketplace dependency.

This page is the shortest path: publish two well-known documents, copy-paste an Ed25519 signer, and verify against existing conformance vectors.

Quickstart

  1. Publish https://<your-domain>/.well-known/vacation-rental.json with protocol, protocol_version, canonical_domain, jwks_url, and verified_stay_offer_endpoint.
  2. Publish https://<your-domain>/.well-known/jwks.json with your Ed25519 public key (kty: OKP, crv: Ed25519, x, kid).
  3. Use the signer below in your offer endpoint to emit compact JWS in signature.jws.
  4. Return signed true/false availability (including explicit signed unavailable) and fail closed on missing or invalid requirements.
  5. Run conformance vectors and self-verify in browser: /verify.

Copy-paste Ed25519 signer (Node.js)

Minimal signer for a VRP verified_stay_offer envelope. Keep private keys server-side only. 

import { createPrivateKey, sign } from "node:crypto";

const b64url = (input) => Buffer.from(input).toString("base64url");
const encodeJson = (value) => b64url(JSON.stringify(value));

export function signVerifiedStayOffer({ offer, privateJwk, kid }) {
  const header = { alg: "EdDSA", kid, typ: "JOSE" };
  const protectedHeader = encodeJson(header);
  const payload = encodeJson(offer);
  const signingInput = `${protectedHeader}.${payload}`;

  const key = createPrivateKey({ key: privateJwk, format: "jwk" });
  const signature = sign(null, Buffer.from(signingInput), key);
  const jws = `${signingInput}.${b64url(signature)}`;

  return {
    ...offer,
    signature: { alg: "EdDSA", kid, jws },
  };
}
Endpoint sketch: GET /api/verified-stay-offer?check_in=YYYY-MM-DD&check_out=YYYY-MM-DD&guests=2 -> build offer payload -> sign with signVerifiedStayOffer() -> return JSON.
